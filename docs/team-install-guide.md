# MD-Browser 团队内测安装与首次配置

这份说明面向只需要使用客户端的团队成员，不需要打开源码，也不需要运行命令行开发脚本。

## 你会收到什么

- 安装包：`MD-Browser-0.2.0-arm64.dmg`
- 版本清单：`MD-Browser-latest-mac-arm64.json`，负责人用于核对版本和文件校验，普通成员通常不需要打开
- 适用设备：Apple Silicon / M 系列 Mac
- 当前版本：`0.2.0`
- 文件校验：983b9f9e7c0c27b56ea4dfadf6834b6a7cfa206859987048a11dbcc360e38acd

当前是未签名内测包，只适合内部验证。正式分发版本会做 Apple 签名和公证。

## 安装

1. 双击打开 `MD-Browser-0.2.0-arm64.dmg`。
2. 把 `MD-Browser.app` 拖到 `Applications`。
3. 打开 `Applications` 里的 `MD-Browser`。

如果 macOS 提示“无法打开”：

1. 在访达里进入 `Applications`。
2. 找到 `MD-Browser.app`。
3. 右键点击，选择“打开”。
4. 在弹窗里再次确认“打开”。

如果仍然被拦截，联系负责人处理。内测阶段可以用下面命令解除隔离标记：

```bash
xattr -dr com.apple.quarantine /Applications/MD-Browser.app
open /Applications/MD-Browser.app
```

## 首次打开后做什么

### 1. 选择代理服务

进入左侧“设置”。

如果你已经在用 Clash Verge Rev：

1. 选择“外部代理客户端”。
2. 点击“选择客户端”。
3. 选择检测到的 Clash Verge Rev。
4. 保存设置。

如果你想让 MD-Browser 自己管理代理：

1. 选择“内置 Mihomo”。
2. 填写你的代理订阅地址。
3. 点击“安装并启用”。

内置模式会自动保存设置、把代理来源切换为内置、安装 Core、生成配置并启动代理。节点页面只负责展示当前代理来源和节点池，不负责安装或切换代理模式。

### 2. 确认目录池

进入左侧“目录池”。

MD-Browser 会自动扫描本机常见 Chrome 资料目录。如果没有看到你要用的浏览器资料，可以点击导入目录池。

目录池只保存在你自己的电脑上，不会上传账号登录态。

### 3. 创建浏览器配置

进入左侧“浏览器配置”。

点击添加按钮，填写：

- 配置名称，例如 `TikTok US 主号`
- 绑定节点
- 默认启动网址，可不填

CDP 端口和代理入口端口会自动分配，通常不需要手动改。

### 4. 启动浏览器

在浏览器配置列表里点击启动按钮。

启动后可以确认：

- 状态显示可用
- 浏览器窗口打开
- 目标网站可以访问
- 配置里的节点和代理入口正常

## 团队配置导入

如果负责人给了团队配置 JSON：

1. 进入“浏览器配置”。
2. 点击导入按钮。
3. 粘贴 JSON。
4. 预览后确认导入。

导入后如果节点不存在，说明你的代理订阅里没有同名节点，需要自己重新绑定一个可用节点。

团队配置 JSON 只共享配置名称、默认网址和期望节点。导入时，CDP 端口、代理入口端口、`user-data-dir` 和 Chrome Profile 会在当前电脑重新分配，不会带入负责人或其他同事的账号登录态。

## Agent / Codex 使用方式

MD-Browser 会把浏览器配置提供给 Codex / Agent。

如果团队成员使用 Codex，推荐启用 `MD-Browser` 插件。插件会连接本机 MD-Browser App，读取当前电脑上的浏览器配置。

Agent 可以读取：

- 配置名称
- CDP 端口
- 代理入口
- 绑定节点
- 默认启动网址

使用时不需要手动告诉 Agent 每个端口对应什么网站，让 Agent 读取 MD-Browser 配置即可。

常用说法：

- 用 MD-Browser 查看当前浏览器配置
- 用 MD-Browser 启动某个浏览器配置
- 用 MD-Browser 在指定配置里打开网站

注意：Codex 插件本身不保存账号登录态，也不替代 MD-Browser App。浏览器配置、目录池、代理模式和节点绑定仍然在 MD-Browser App 里维护。

## 常见问题

### 打开 App 后页面空白或无法连接

退出 MD-Browser 后重新打开。如果仍然失败，联系负责人并提供当前版本号。

版本号可以在“设置”里的本地服务卡片看到。配置文件、脚本日志和内置 Mihomo 路径可以在“设置”的诊断信息卡片看到。

### 浏览器启动了，但网站打不开

优先检查：

- 节点是否可用
- 代理入口是否在线
- 当前订阅里是否还有绑定的节点

如果切换过代理订阅，原来绑定的节点可能已经不存在，需要重新选择节点。

### 可以共用别人的登录态吗

不建议。MD-Browser 的设计是每个人使用自己电脑上的 `user-data-dir` 和 Chrome Profile，账号登录态只保存在本机。

### 升级会清空配置吗

不会。App 配置保存在：

```text
~/.md-browser/config.json
```

浏览器登录态保存在你选择的 Chrome 资料目录里。升级 App 不会删除这些内容。
