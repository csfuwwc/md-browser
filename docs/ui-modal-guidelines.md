# UI 弹窗与页面动作规范

适用范围：

- 添加浏览器配置
- 添加目录池
- 编辑配置
- 运行日志
- 后续所有导入、选择、确认类弹窗

目标：

- 统一结构
- 统一间距
- 统一信息层级
- 降低临时性样式调整

## 1. 基本结构

所有弹窗固定分为三段：

1. 头部
2. 内容区
3. 底部操作区

不要增加多余层级，也不要把说明文案游离放置。

## 2. 头部规范

头部只负责标识当前动作。

保留：

- 主标题
- 右上角关闭按钮

不保留：

- 英文副标题
- 冗余说明句
- 重复解释

要求：

- 标题字号统一
- 关闭按钮统一放在右上角，使用现有 `icon-button` 样式
- 关闭按钮与取消按钮语义一致，都是关闭当前弹窗，不提交内容
- 头部与内容区之间固定留白
- 头部高度不要因为文案长短发生明显变化

## 3. 内容区规范

内容区只放用户当前要操作的内容。

字段统一采用以下结构：

1. 标题行
2. 输入控件
3. 错误提示

即：

- 字段名 + 右侧弱提示
- 输入框 / 下拉框 / 列表
- 仅错误时显示红字

## 4. 字段标题行规范

字段标题行统一为左右结构：

- 左侧：字段名
- 右侧：弱化说明

适合放在右侧弱提示的信息：

- 自动分配，可手动修改
- 从已导入目录中选择
- 首次启动会自动使用或创建 Default
- 已检测到 Default / Profile 1 / Profile 2

不适合放在右侧弱提示的信息：

- 长段说明
- 操作步骤
- 与字段无直接关系的背景解释

## 5. 字段下方提示规范

字段下方只保留错误提示。

默认状态：

- 不显示绿色帮助说明
- 不显示成功提示
- 不保留明显占位

错误状态：

- 在字段下方显示红字
- 只描述当前字段的问题

顶部错误条只用于：

- 接口失败
- 保存失败
- 系统异常

## 6. 间距规范

弹窗内部统一使用固定节奏，不按单个弹窗临时调整。

要求：

- 头部 -> 内容区：固定间距
- 字段块 -> 字段块：固定间距
- 分段切换 -> 下方内容：固定间距
- 内容区 -> 底部操作区：固定间距
- 底部操作区自身内边距固定

原则：

- 同层级同间距
- 不允许同一弹窗内混用多组近似数值

## 7. 底部操作区规范

底部操作区只放按钮。

要求：

- 与内容区有明确留白
- 建议带浅分隔线
- 左侧放次操作
- 右侧放主操作

典型按钮布局：

- 左：取消
- 右：保存 / 导入 / 确认

## 8. 尺寸规范

弹窗宽度统一分三档：

- 小弹窗：单输入、确认类
- 中弹窗：标准表单类
- 大弹窗：列表选择、日志、复杂查看类

不要为单个弹窗随意定义新的宽度规则。

## 9. 视觉层级规范

信息层级固定为：

1. 标题
2. 字段名
3. 右侧弱提示
4. 输入控件
5. 字段错误提示
6. 顶部系统错误条

不要让弱提示、帮助文案、错误提示同时争抢注意力。

## 10. 页面动作按钮规范

页面右侧或模块操作区的图标按钮统一使用现有动作按钮体系。

刷新类按钮：

- 视觉上与同区域主操作按钮保持一致，使用 `action-tool-button action-tool-button-primary`
- 只通过图标区分功能，刷新使用 `refresh-cw` 图标
- 鼠标悬浮显示弱提示文案，例如“重新扫描”或“刷新状态”
- 执行中使用旋转动画表达状态
- 执行中禁用按钮，避免重复触发
- 如果刷新太快完成，动画至少保留一个可感知时长，当前目录池扫描为 3 秒

同一区域存在多个主要动作时：

- 按钮尺寸、背景、圆角、阴影保持一致
- 不通过不同颜色区分同级动作
- 通过图标和悬浮提示说明功能
- 危险动作例外，继续使用删除图标的红色悬浮反馈

## 11. 当前项目执行规则

从本规范建立后，后续弹窗默认按以下方式实现：

- 头部只保留主标题
- 默认包含右上角关闭按钮
- 字段标题行采用“左标题 + 右弱提示”
- 字段下方只保留红字错误
- 默认不显示绿色说明
- 底部操作区统一分隔与留白
- 单列布局优先
- 后续刷新按钮默认采用主操作图标按钮风格，并带旋转状态
