# MD-Browser Roadmap

## v0.1.0 - Local MVP

目标：把“浏览器端口 + 代理入口 + Profile + 节点”的核心链路跑通，供内部验证。

范围：

- 本地可视化配置中心。
- 浏览器配置、目录池、节点池、系统设置、运行日志。
- 外部代理客户端和内置 Mihomo 两种代理模式。
- 配置导入导出，支持团队共享浏览器配置模板。
- HTTP MCP，让 Codex / Agent 能读取配置并打开指定浏览器。
- Mac arm64 内测包构建。
- 默认 Chrome 保护：不复用默认浏览器资料目录，不接管默认浏览器端口。

验收：

- 新建浏览器配置后，可以用独立身份启动指定 CDP 端口。
- 绑定节点后，代理入口能按配置生效。
- Agent 能读取配置并启动或连接指定浏览器。
- `npm test` 全量通过。

## v0.2.0 - Team Beta

目标：让非技术同事可以稳定安装、首次配置和日常使用。

范围：

- 首次使用流程产品化：代理服务、目录池、浏览器配置、Agent 接入按步骤引导。
- 外部代理客户端选择流程完善：明确 Clash Verge Rev 检测、连接状态、失败原因和修复入口。
- 内置 Mihomo beta：订阅配置、Core 下载、启动、停止、一键修复更稳定。
- 节点可用性检测：绑定节点失效、订阅切换导致节点不存在、节点延时异常时给出可操作提示。
- 浏览器配置诊断闭环：端口占用、代理离线、节点无效、浏览器未启动都在页面内给出下一步。
- Apple Developer 签名和 notarization 打通，产出可直接安装的团队内测包。
- GitHub Releases 作为版本分发入口，维护 release notes 和安装包下载。

验收：

- 新同事拿到安装包后，不看技术文档也能完成代理连接、创建配置、启动浏览器。
- 外部代理和内置代理都能明确显示“可用 / 不可用 / 修复中 / 已运行”。
- v0.1.0 配置升级到 v0.2.0 不丢失本地配置和登录态。

## v0.3.0 - Managed Team Workflow

目标：从单机配置工具升级为团队可复制的浏览器路线工作流。

范围：

- 团队模板能力增强：按业务场景导出、导入、覆盖、合并和版本化配置模板。
- 更新机制：客户端启动时检查远端 release manifest，提示可升级版本。
- Agent 接入增强：提供更稳定的配置查询、启动、打开网址、状态检测和错误说明。
- 节点策略增强：支持订阅切换后的节点重绑定建议、批量检测和批量修复。
- 操作日志和排障包：一键导出配置摘要、运行日志和环境状态，便于负责人远程排障。
- 权限和安全边界明确：不导出敏感信息，不共享 Cookie，不修改默认浏览器。
- 自动更新方案评估：在签名、公证和回滚策略稳定后再接入自动更新。

验收：

- 负责人可以维护一份团队模板，新成员导入后只需要选择本机代理和账号身份。
- Agent 能稳定按配置名打开对应浏览器，并能解释失败原因。
- 升级 App 不破坏用户本机端口、节点、目录池和登录态。

## 版本规则

- `0.x`：内部测试和团队 beta，允许快速调整 UI 与字段，但必须保护本地配置和登录态。
- `1.0.0`：配置结构稳定，团队可正式使用。
- `1.x`：只做兼容升级，不破坏旧配置。
- `2.0.0`：如需破坏性配置变更，必须提供迁移、备份和回滚方案。
