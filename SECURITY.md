# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive findings.

Report security issues privately to the repository maintainers with:

- A short description of the issue
- Impact and affected area
- Reproduction steps
- Any suggested mitigation

Until a dedicated security contact is published, coordinate through a private maintainer channel rather than a public GitHub issue.

## Scope

Security-sensitive areas in this project include:

- Local configuration storage
- Browser profile isolation
- Proxy controller credentials
- Support bundle sanitization
- MCP and local HTTP control endpoints
